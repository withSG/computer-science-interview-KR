# 인성/경험 면접 질문 & 답변

## 사용 방법
1. 질문을 먼저 읽고 자신만의 경험으로 답변을 준비하세요
2. STAR 기법을 활용해 구조화된 답변을 만드세요
3. 실제 경험 기반으로 진정성 있게 답변하세요

---

## STAR 기법

**S**ituation (상황): 어떤 상황이었나요?
**T**ask (과제): 무엇을 해야 했나요?
**A**ction (행동): 어떻게 행동했나요?
**R**esult (결과): 어떤 결과가 있었나요?

## Q1. 자기소개를 해주세요. ⭐⭐⭐

<details>
<summary>답변 가이드</summary>

### 구조

```
1. 간단한 인사 + 이름
2. 핵심 역량/경험 (30초)
3. 지원 포지션과의 연결 (20초)
4. 마무리 인사
```

### 예시 프레임

```
안녕하세요, [이름]입니다.

저는 [N년]간 [분야]에서 [역할]을 수행했습니다.
특히 [주요 프로젝트/성과]를 통해 [역량]을 쌓았습니다.

[회사/팀]의 [비전/프로젝트]에 관심이 있어 지원했으며,
[기여할 수 있는 점]으로 팀에 기여하고 싶습니다.
```

### 주의사항

- 1-2분 내로 간결하게
- 이력서 나열 X, 핵심만
- 지원 포지션과 연결
- 자신감 있게

</details>

## Q2. 왜 이 회사에 지원했나요? ⭐⭐⭐

<details>
<summary>답변 가이드</summary>

### 포인트

```
1. 회사에 대한 관심 (구체적 리서치)
2. 나의 역량과 연결
3. 성장 가능성
```

### 예시 프레임

```
[회사]의 [특정 서비스/기술/문화]에 관심을 갖게 되었습니다.

특히 [구체적 프로젝트/뉴스]를 보고
제가 [경험/역량]으로 기여할 수 있겠다고 생각했습니다.

또한 [회사의 성장/문화]를 통해 함께 성장하고 싶습니다.
```

### 준비사항

- 회사 최근 뉴스, 블로그 확인
- 서비스 직접 사용 경험
- JD(Job Description) 분석

</details>

## Q3. 본인의 강점과 약점은? ⭐⭐⭐

<details>
<summary>답변 가이드</summary>

### 강점

```
1. 강점 명시
2. 증거 (구체적 경험)
3. 어떻게 활용할지
```

### 약점

```
1. 진짜 약점 (하지만 치명적이지 않은)
2. 개선 노력
3. 발전 과정
```

### 예시

```
강점: "저는 문제 해결력이 강점입니다.
이전 프로젝트에서 성능 이슈가 있었는데,
원인을 분석하고 [해결책]으로 [결과]를 달성했습니다."

약점: "꼼꼼함이 과할 때가 있어 속도가 느려질 수 있습니다.
이를 개선하기 위해 타임박싱을 적용하고,
우선순위에 따라 완성도를 조절하려 노력합니다."
```

### 피해야 할 답변

- "약점이 없습니다" (오만)
- "완벽주의입니다" (진부)
- 포지션에 치명적인 약점

</details>

## Q4. 가장 도전적이었던 프로젝트는? ⭐⭐⭐

<details>
<summary>답변 가이드 (STAR 활용)</summary>

### Situation (상황)

```
"[시점]에 [프로젝트명]을 진행했습니다.
[규모/팀 구성]으로, [목표]를 달성해야 했습니다."
```

### Task (과제)

```
"제 역할은 [담당 업무]였고,
특히 [도전적인 부분]이 있었습니다."
```

### Action (행동)

```
"저는 [구체적 행동 1]을 했습니다.
또한 [구체적 행동 2]를 통해 문제를 해결했습니다."
```

### Result (결과)

```
"결과적으로 [정량적 성과]를 달성했고,
[정성적 성과/배움]을 얻었습니다."
```

### 예시

```
S: "작년에 실시간 채팅 기능을 개발했습니다.
    트래픽이 높은 서비스여서 성능이 중요했습니다."

T: "저는 백엔드 설계와 WebSocket 구현을 담당했고,
    동시 접속 1만 명을 처리해야 했습니다."

A: "Redis Pub/Sub으로 메시지 브로커를 구성하고,
    Connection Pool을 최적화했습니다.
    부하 테스트를 통해 병목점을 찾아 개선했습니다."

R: "목표 대비 150% 성능을 달성했고,
    실시간 시스템 설계 역량을 키웠습니다."
```

</details>

## Q5. 협업 중 갈등 상황을 어떻게 해결했나요? ⭐⭐

<details>
<summary>답변 가이드</summary>

### 핵심 포인트

```
1. 갈등 상황 객관적 설명
2. 상대방 입장 이해 노력
3. 건설적 해결 과정
4. 결과와 배움
```

### 예시 프레임

```
S: "프론트엔드 개발자와 API 스펙에 대해 의견 차이가 있었습니다."

T: "양측 모두 합리적 이유가 있었고, 합의가 필요했습니다."

A: "먼저 상대방 관점을 경청하고 이유를 파악했습니다.
    데이터를 기반으로 각 방식의 장단점을 비교했고,
    절충안을 제안하여 함께 결정했습니다."

R: "합의된 방식으로 개발을 완료했고,
    이후 비슷한 상황에서 참고할 가이드라인을 만들었습니다."
```

### 주의사항

- 비난하지 않기
- 해결 과정에 초점
- 배운 점 강조

</details>

## Q6. 실패 경험과 그것에서 배운 점은? ⭐⭐

<details>
<summary>답변 가이드</summary>

### 핵심 포인트

```
1. 실패 인정 (숨기지 말기)
2. 원인 분석
3. 개선 행동
4. 성장한 점
```

### 예시

```
S: "첫 프로젝트에서 일정 지연이 발생했습니다."

T: "예상보다 복잡한 요구사항을 과소평가했습니다."

A: "일정 지연 원인을 분석하고,
    이후 프로젝트에서는 버퍼 시간을 두고
    주간 진척 체크를 도입했습니다."

R: "이후 프로젝트는 일정 내 완료했고,
    초기 기획 단계의 중요성을 배웠습니다."
```

</details>

## 면접 팁

### Before

```
- 회사/팀 리서치
- JD 분석, 나의 경험 매칭
- 모의 면접으로 연습
- 질문 리스트 준비
```

### During

```
- 경청하고 질문 의도 파악
- STAR로 구조화
- 모르면 솔직히 말하기
- 역질문 준비
```

### After

```
- 감사 메일 (선택)
- 피드백 요청
- 복기 및 개선
```
