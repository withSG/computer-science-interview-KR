## Q1. 과적합(Overfitting)이란 무엇이며, 어떻게 완화하나요? ⭐⭐⭐

<details>
<summary>답변 보기</summary>

### 핵심 답변
과적합은 **모델이 학습 데이터에는 잘 맞지만 새로운 데이터(테스트)에는 일반화하지 못하는** 현상입니다. 모델이 데이터의 패턴이 아니라 노이즈까지 외워버린 상태입니다.

### 과적합 vs 과소적합

| 구분 | 과소적합(Underfitting) | 과적합(Overfitting) |
|------|------------------------|---------------------|
| 학습 오차 | 높음 | 낮음 |
| 검증 오차 | 높음 | 높음 |
| 원인 | 모델이 너무 단순 | 모델이 너무 복잡 |
| 비유 | 공부 안 함 | 답지만 통째로 암기 |

```
오차
 │        검증 오차
 │       ╱
 │      ╱   ← 과적합 구간 (검증 오차 다시 증가)
 │\    ╱
 │ \  ╱
 │  \╱
 │   학습 오차 (계속 감소)
 └──────────────── 모델 복잡도/학습량
   적정 지점(generalization gap 최소)
```

### 완화 방법

```
1. 데이터 늘리기 / 데이터 증강(Augmentation)
2. 정규화 (L1/L2, Dropout)
3. 모델 단순화 (파라미터 수 축소)
4. 조기 종료 (Early Stopping)
5. 교차 검증으로 일반화 성능 확인
6. (딥러닝) Batch Normalization
```

### 면접관이 주목하는 포인트
- 학습/검증 오차의 격차(generalization gap)로 진단
- 완화 방법을 여러 개 제시

</details>

## Q2. L1 정규화와 L2 정규화의 차이는? ⭐⭐⭐

<details>
<summary>답변 보기</summary>

### 핵심 답변
정규화(Regularization)는 **손실 함수에 가중치 크기에 대한 페널티를 추가**해 모델이 너무 복잡해지는 것을 막는 기법입니다. L1은 가중치 절댓값 합, L2는 가중치 제곱 합을 페널티로 씁니다.

### 비교

| 구분 | L1 (Lasso) | L2 (Ridge) |
|------|-----------|-----------|
| 페널티 | λ·Σ\|w\| | λ·Σw² |
| 효과 | 일부 가중치를 **0으로** | 가중치를 **작게** (0은 잘 안 됨) |
| 특성 선택 | 가능 (희소성) | 불가 |
| 사용처 | 특성이 많고 일부만 중요할 때 | 일반적인 과적합 방지 |

### 수식

```
손실 = 원래 손실 + λ · 페널티

L1: Loss + λ Σ|wᵢ|        → 희소(sparse) 해, 특성 선택 효과
L2: Loss + λ Σwᵢ²         → 가중치를 전반적으로 축소(weight decay)

λ(람다): 정규화 강도. 크면 규제↑(과소적합 위험), 작으면 규제↓(과적합 위험)
```

### 왜 L1은 0을 만드는가
- L1의 페널티 등고선은 마름모(꼭짓점이 축 위)라서 최적해가 축(=가중치 0)에서 만나기 쉽습니다. L2는 원형이라 0이 잘 되지 않습니다.

### Dropout (딥러닝의 정규화)
- 학습 시 뉴런 일부를 무작위로 끄는 기법. 특정 뉴런에 과의존하지 못하게 해 과적합을 완화합니다. 추론 시에는 모든 뉴런을 사용합니다.

### 면접관이 주목하는 포인트
- L1의 희소성(특성 선택) vs L2의 가중치 축소
- 정규화 강도 λ의 트레이드오프

</details>

## Q3. 전이학습(Transfer Learning)이란 무엇인가요? (ResNet 사례) ⭐⭐

<details>
<summary>답변 보기</summary>

### 핵심 답변
전이학습은 **대규모 데이터로 미리 학습된 모델(pre-trained)을 가져와, 우리 과제에 맞게 재사용**하는 기법입니다. 처음부터 학습하지 않으므로 적은 데이터와 적은 비용으로 좋은 성능을 낼 수 있습니다.

### 왜 효과적인가

```
ImageNet(100만+ 이미지)로 학습된 ResNet18은
이미 "엣지 → 질감 → 형태" 같은 일반적 시각 특징을 학습함.
→ 이 특징 추출기를 재사용하면, 우리 데이터가 적어도 잘 동작.
```

### 적용 방식

| 방식 | 설명 | 적합한 상황 |
|------|------|------------|
| Feature Extraction | 사전학습 가중치 **고정(freeze)**, 마지막 분류층만 새로 학습 | 데이터가 적을 때 |
| Fine-tuning | 일부/전체 층의 가중치를 **추가 학습** | 데이터가 어느 정도 있을 때 |

### ResNet18 미세조정 예시 (PyTorch)

```python
import torch.nn as nn
from torchvision import models

# 1. 사전학습된 ResNet18 로드
model = models.resnet18(weights="IMAGENET1K_V1")

# 2. 특징 추출기는 고정
for param in model.parameters():
    param.requires_grad = False

# 3. 마지막 분류층을 내 과제(클래스 수=10)로 교체
model.fc = nn.Linear(model.fc.in_features, 10)

# 이제 model.fc만 학습됨 (전이학습)
```

> ResNet의 핵심은 **잔차 연결(Residual/Skip Connection)** 으로, 층이 깊어져도 기울기 소실 없이 학습되게 합니다.

### 면접관이 주목하는 포인트
- 전이학습이 적은 데이터에서 유리한 이유
- Feature Extraction vs Fine-tuning 선택 기준

</details>

## Q4. 분류 모델의 평가지표를 설명해주세요. ⭐⭐⭐

<details>
<summary>답변 보기</summary>

### 핵심 답변
정확도(Accuracy) 하나로는 부족하며, **혼동 행렬(Confusion Matrix)** 기반으로 정밀도/재현율/F1 등을 함께 봐야 합니다. 특히 데이터가 불균형하면 정확도는 오해를 부릅니다.

### 혼동 행렬

```
                예측 Positive   예측 Negative
실제 Positive      TP              FN
실제 Negative      FP              TN
```

### 주요 지표

| 지표 | 공식 | 의미 |
|------|------|------|
| Accuracy | (TP+TN) / 전체 | 전체 중 맞춘 비율 |
| Precision | TP / (TP+FP) | 양성 예측 중 진짜 양성 비율 |
| Recall | TP / (TP+FN) | 실제 양성 중 잡아낸 비율 |
| F1 | 2·P·R / (P+R) | 정밀도와 재현율의 조화평균 |

### 정밀도 vs 재현율 트레이드오프

```
스팸 필터:  Precision 중요 (정상 메일을 스팸으로 막으면 안 됨)
암 진단:    Recall 중요 (실제 환자를 놓치면 안 됨)
```

### 정확도의 함정 (불균형 데이터)

```
환자 1%, 정상 99%인 데이터에서
"전부 정상"이라고 예측 → Accuracy 99% (그러나 환자는 0명 검출)
→ 이때는 Recall, F1, ROC-AUC를 봐야 함
```

### ROC-AUC
- 임계값을 변화시키며 TPR(재현율) vs FPR을 그린 곡선의 아래 면적(AUC). 1에 가까울수록 좋고, 0.5는 무작위 수준입니다. 임계값에 독립적인 종합 성능 지표입니다.

### 회귀 지표 (참고)

| 지표 | 의미 |
|------|------|
| MSE | 오차 제곱 평균 (큰 오차에 민감) |
| MAE | 오차 절댓값 평균 (이상치에 덜 민감) |
| R² | 모델이 설명하는 분산 비율 (1에 가까울수록 좋음) |

### 면접관이 주목하는 포인트
- 불균형 데이터에서 정확도의 한계
- 도메인에 따른 정밀도/재현율 선택

</details>

## Q5. 편향-분산 트레이드오프와 교차 검증을 설명해주세요. ⭐⭐

<details>
<summary>답변 보기</summary>

### 핵심 답변
모델의 일반화 오차는 **편향(Bias) + 분산(Variance) + 줄일 수 없는 노이즈**로 분해됩니다. 편향과 분산은 서로 상충(트레이드오프)하므로 둘의 합이 최소가 되는 균형점을 찾아야 합니다.

### 편향 vs 분산

| 구분 | 편향(Bias) | 분산(Variance) |
|------|-----------|----------------|
| 의미 | 예측이 정답에서 체계적으로 벗어남 | 데이터가 바뀌면 예측이 크게 흔들림 |
| 관련 | 과소적합 | 과적합 |
| 모델 | 너무 단순 | 너무 복잡 |

```
높은 편향:  과녁 한쪽에 몰려 빗나감 (일관되게 틀림)
높은 분산:  과녁에 넓게 흩어짐 (불안정)
목표:       낮은 편향 + 낮은 분산
```

### 교차 검증 (Cross Validation)

데이터를 여러 폴드로 나눠 번갈아 검증해, **일반화 성능을 안정적으로 추정**합니다.

```
K-Fold (K=5) 예시:
[검증][학습][학습][학습][학습]
[학습][검증][학습][학습][학습]
[학습][학습][검증][학습][학습]
...  → 5번의 성능을 평균
```

- 데이터가 적을 때 특히 유용하며, 단일 train/test 분할의 운(偶然)에 덜 휘둘립니다.

### 면접관이 주목하는 포인트
- 편향↔분산이 과소적합↔과적합과 연결됨
- 교차 검증이 일반화 성능을 더 신뢰성 있게 추정하는 이유

</details>

## Q6. 설명 가능한 AI(XAI)란 무엇이며, SHAP은 무엇인가요? ⭐⭐

<details>
<summary>답변 보기</summary>

### 핵심 답변
XAI(eXplainable AI)는 **모델이 왜 그런 예측을 했는지 사람이 이해할 수 있게 설명하는** 기법입니다. 딥러닝 등 블랙박스 모델의 신뢰성·공정성·디버깅을 위해 중요합니다.

### 왜 필요한가

```
- 신뢰: 의료/금융 등 고위험 도메인에서 근거 필요
- 규제: 자동 의사결정에 대한 설명 요구 (예: GDPR)
- 디버깅: 모델이 엉뚱한 특징(배경, 편향)을 보는지 점검
- 공정성: 차별적 특징에 의존하는지 확인
```

### 주요 기법

| 기법 | 설명 |
|------|------|
| Feature Importance | 각 특징이 예측에 기여한 정도 (트리 모델 등) |
| SHAP | 게임이론(Shapley value) 기반으로 특징별 기여도를 일관되게 계산 |
| LIME | 예측 주변을 단순 모델로 근사해 국소적으로 설명 |
| Grad-CAM | (CNN) 이미지에서 모델이 주목한 영역 시각화 |

### SHAP (SHapley Additive exPlanations)

```
- 협력 게임 이론의 Shapley value를 차용
- "각 특징을 넣고 빼면서 예측에 얼마나 기여했나"를 공정하게 분배
- 전역(global) 설명 + 개별 예측(local) 설명 모두 가능
- 기여도의 합 = 예측값 - 기준값 (가산적, additive)
```

```python
import shap
explainer = shap.TreeExplainer(model)
shap_values = explainer.shap_values(X)
shap.summary_plot(shap_values, X)  # 특징별 기여도 시각화
```

### 면접관이 주목하는 포인트
- XAI가 필요한 도메인/상황
- SHAP이 특징 기여도를 일관되게 분배한다는 개념

</details>
